Pasos para resolver el problema

Diseño gráfico

Realice un "dibujo" (diseño gráfico) de su sitio. Debe ser personal, es decir, debe representarle simbólicamente. Por ejemplo, si su pasión en la vida es el ajedrez puede utilizar el ajedrez como métafora del diseño. Para la conceptualización de su diseño gráfico puede ayudarse con el libro Principles of Beautiful Web Design.

Traduzca su diseño gráfico a CSS y [X]HTML

Realice una plantilla HTML en base al diseño gráfico. Debe hacer su diagramación con HTML 5 y CSS. No utilice tablas cuando su uso pervierta su semántica. Obligatoriamente debe utilizar el framework para front-end Bootstrap. Su diseño debe ser Mobile First y debe verse bien tanto en un dispositivo móvil como en escritorio. La plantilla debe ser lo más semántica posible. Para ello debe utilizar preprocesadores CSS. La presentación debe estar separada al máximo de su plantilla de manera tal que si se cambia su CSS es posible verla con otro diseño gráfico.

Cree las páginas estáticas

Todas las páginas deben tener el contenido pertinente. No haga un sitio de una página (que simule varias). Puede utilizar editores WYSIWYG como Dreamweaver (propietario), BlueGriffon (libre) o Visual Studio Code (gratuito). Se recomienda utilizar Visual Studio Code.

Modularice

Además deberá utilizar Server Side Includes para que cada archivo en realidad sólo sea contenido y se facilite el mantenimiento.
